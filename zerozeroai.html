Zero Zero AI

Created June 2026, by Jorge Franco

In 007 Skyfall, there is a conversation between Bond and the new Q:

Q: I can do more damage on my laptop sitting in my pijamas before my first cup of Earl Grey than you can do in a year in the field.

Bond: Oh, so why do you need me?

Q: Every now and then, a trigger has to be pulled.

Bond: Or not pulled... It's hard to know which in your pijamas.

To me, this is a fantastic example of seniority, which applies very well to my profession as a software engineer, and also relates to the current situation of the profession with AI adoption. Because it's important to know what this profession is, starting with the name: coder, developer, programmer,... I think these days, when AI is trending, software engineer fits better, because it's really what companies are paying for, in the past, now, and in the future.

When I started my career, writing as much code as possible was my target. At the beginning, I just wanted to do as much as possible, learning by doing, ignoring the future of code created. And it's good; you need to learn, you have to fail. Leaving the "juniority" means spending time getting experience. While you are getting experience, and you start to revisit your code, or code created by others, you start to discover the important topics. Testing, readability, coupling, complexity, domain, people, maintenance, continuous delivery, migrations, performance,... are a few of them. You start to realize that coding itself is not so important, and you become a software engineer.

Now, let's see what AI can do, apart from telling you how amazing your questions are. AI can give you a response, or provide code, from a lot of data in a model, your input, and context. It can do pretty well there, and it will improve over time. I'm very impressed with the results. But, you see, it's just code; for sure a lot better than the code I wrote at the beginning of my career, but nothing more than code, a very small part of what it means to be a software engineer. So here AI provides the first zero, giving just code and stopping real people from growing.

To be a good senior engineer it is not easy. It requires that you have good skills, that you meet people to learn from, work on different projects and technologies, and never stop learning. You need good luck and critical thinking,... and many more things. That is the reason some companies pay a lot of money to get the best software engineers. Maybe you think that these engineers write a lot of code, but you can imagine that it's not true. And it's not true now with AI, but also in the past when you had to write complex code by hand. So, there are a few good software engineers, and they don't write too much code; they design, think, plan, and guide,... many important things that are reflected in just a few lines of code.

And we have AI, which gets the output from a lot of code in the LLM, lines of code converted in columns in tables. There is no real thinking, experience applied, domain knowledge, creativity, standards, evolution or maintenance in the process of generating the code. You can imagine that each day, it will be harder to get code with real value, as more AI generated code will be added to the models. All the software that you can create with AI is just code that anyone can create without effort, a huge amount of code. So, do you really think that code created by AI can be profitable? Maybe, but very difficult, depending on the idea, not the code. Congrats AI for your second zero: you are providing a false feeling of progress, building something with a very low chance of success, full of repeated code and difficult to maintain.

If you are a good software engineer, congratulations, AI will amplify you. If you are not there yet, take care, double zero AI is just a tool that will replace you if you use it extensively.

Coming back to James Bond: the world still requires people like him, a lot of them, people able to decide how and what to do or what to avoid. Bond's software engineers are needed, and I think they will be needed for many years. I have the same feelings of the film scene, innovation is coming to replace us, but, by now, it requires us to create real value. Our profession is changing, making the act of writing code obsolete, but as you've read, it's a small part of what we do.
